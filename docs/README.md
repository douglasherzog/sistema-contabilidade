# Documentação

Este diretório concentra a documentação do projeto.

## Padrão do projeto

- Toda mudança deve ser documentada nos mínimos detalhes.
- Toda implementação deve ser testada.

## Onde registrar mudanças

- `docs/changes/` (um arquivo por mudança/feature/bugfix)

## Índice

- `docs/workflow.md` (workflow e checklist)
- `docs/testing.md` (como validar e rodar testes)
- `docs/ux.md` (diretrizes de interface didática e auto-explicativa)
